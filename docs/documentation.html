Syntax

The syntax is quite understandable and simple, by the way it is similar to the CSS syntax

To use any statement, you must first enter it, let's say bl_start, then if the statement has values necessary for operation, enter a colon (:), and enter the value itself, in our case any character

In general, we get the following statement: bl_start: a but to alternate our statements we must also put a semicolon (;)

You will also see all the code completely below the input line, so that you don't get lost and write everything correctly

Syntax Sugar

There is not much syntactic sugar here, but it is there, the easiest thing is not needing spaces, i.e. statement:value; and statement : value ; will be the same, use it the way you want

There is also another advantage, we can list all values for statement in one line without any delimiters

Personally my favorite way of writing is the same as the basic one in CSS, i.e. statement: value; this is probably the official kind of formatting

Blacklist

Finally we come to the statements themselves, let's look at the most basic and useful part, the blacklist

bl_start

bl_start: a; it forbids us to insert the letter a into the string, which gives us full control over the beginning, what letters we want to see at the beginning of our string and what letters we don't want to see

Yes, that's all, but it gives us a huge advantage for creating some specific strings if used properly

bl_end

bl_end: a; is actually the same, but as you can see from the statement itself, it forbids using values as the end of a string

bl_other

bl_other: a; also performs the same actions as the previous two statements, but only inside the string without touching its edges

bl_all

bl_all: a; combines all of its earlier brethren and removes any character from the string

This is actually the magic of the blacklist, to forbid characters in certain places

Other Statements

Also, there are some unsorted statements here that may help you in some way when creating strings

uc_first

uc_first needs no values, so it can simply be used as uc_first;

It makes the first character always in Uppercase, but also note that if you enable Uppercase Characters you probably won't notice any difference

help (DEPRECATED)

help like uc_first doesn't need a value, so you can just write help;

help opens this documentation, in which you can find many discoveries about this site
